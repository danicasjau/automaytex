AUTOMAYTEX
Automated Bake–Diffuse–Reproject Texturing Pipeline for Autodesk Maya 2026
Author Daniel Casadevall Jauhiainen Platform Autodesk Maya 2026 / 2025 Version 1.0-alpha Renderer Arnold API (Python) Format Maya command API Language Python 3.10+
Abstract
Automaytex is a Maya plugin that automates AI-driven surface texture generation from within the authoring environment. The system encodes geometric information — surface normals and depth — from four optimally-placed cameras using a tetrahedral projection strategy, composites this into a 2×2 image atlas, and conditions a Diffusion Model (SDXL, Flux, or SD1.5) via ControlNet to produce a spatially-coherent texture. The result is reprojected onto the original UV layout using depth-weighted seam blending. Hardware-aware quantization ensures the pipeline adapts to available GPU memory without manual configuration.

1. Overview

Automaytex targets look-development artists who need to rapidly prototype surface materials on 3D assets without leaving Maya. The plugin wraps a complete AI inference pipeline — geometry extraction, atlas compositing, diffusion-model inference, and UV reprojection — behind a single shelf button and a minimal panel.

The workflow assumes the mesh already carries a valid, non-overlapping UV layout. UVs are not generated by the plugin. All intermediate outputs (AOV renders, atlas images, diffusion results) are written to a configurable temp directory and can be inspected independently.

Key Properties

  • Non-destructive: Original UVs are snapshotted before the pipeline starts. The mesh geometry is never modified; only a new shader is applied at the end.
  • Resolution-independent: Atlas resolution and final output resolution are configured separately in config.yaml.
  • Hardware-adaptive: VRAM is queried at startup; model precision is chosen automatically (FP16 / INT8 / INT4).
  • Multi-model: Flux, SDXL, and Stable Diffusion 1.5 share a common inference interface in engine.py.

git clone https://github.com/youruser/automaytex.git

# Always use mayapy, not the system Python interpreter
mayapy -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
mayapy -m pip install diffusers transformers accelerate bitsandbytes pillow

3 — Download model weights

Weights are not included in the repository. Edit config.yaml to point to local checkpoints obtained from HuggingFace Hub or Civitai (see §12).

4 — Load the plugin in Maya

  1. Open Window → Settings/Preferences → Plug-in Manager.
  2. Click Browse and select src/main.py.
  3. Enable Loaded (and optionally Auto load).
  4. The Automaytex shelf button will appear in the Maya toolbar.

4. Pipeline Architecture

The pipeline is divided into three sequential, isolated phases. Each writes its outputs to a temp directory and can be triggered independently from the Script Editor for debugging. A UV snapshot is taken before any render begins.

Pre-step — UV Snapshot

Before any render is triggered, the plugin serialises the mesh UV data to uv_manifest.json in the temp directory. This guarantees that Phase 3 uses the exact UV layout present at generation time, regardless of any subsequent edits.

5. Tetrahedral Viewpoint Strategy

The original implementation used a 6-plane orthographic projection (box mapping: ±X, ±Y, ±Z). This was replaced by a tetrahedral strategy for two reasons: (1) opposing face pairs produce near-duplicate captures for most meshes, wasting inference budget; (2) arranging 6 views into a single atlas produces non-square sub-regions that mismatch Diffusion Model training distributions.

Geometry

A regular tetrahedron inscribed in the mesh's bounding sphere provides four vertex directions that are maximally and uniformly separated. The angular separation between any two vertices is:

(1) $$\theta = \arccos\!\left(-\tfrac{1}{3}\right) \approx 109.47°$$

Placing one camera at each vertex, aimed at the mesh centroid $\mathbf{c}$, ensures near-uniform coverage with minimal redundant overlap (~12% average, versus ~35% for box projection).

Camera Placement

Let $\mathbf{c}$ be the mesh centroid and $r$ the bounding-sphere radius. The four tetrahedral vertex unit directions are:

(2) $$\hat{d}_0 = \begin{pmatrix}0\\1\\0\end{pmatrix}, \quad \hat{d}_1 = \begin{pmatrix}0.943\\-0.333\\0\end{pmatrix}, \quad \hat{d}_2 = \begin{pmatrix}-0.471\\-0.333\\0.816\end{pmatrix}, \quad \hat{d}_3 = \begin{pmatrix}-0.471\\-0.333\\-0.816\end{pmatrix}$$

Each camera is positioned at $\mathbf{p}_i = \mathbf{c} + 2r\,\hat{d}_i$ with a look-at transform aimed at $\mathbf{c}$:

(3) $$\mathbf{M}_i = \mathrm{LookAt}\!\left(\mathbf{p}_i,\; \mathbf{c},\; \hat{u}\right)$$

Atlas Layout

The four camera views are tiled into a 2×2 grid, producing a single square atlas compatible with Diffusion Model training distributions. The same layout is used for both the normals atlas and the depth atlas.

Strategy Views Avg. overlap Atlas sub-regions
Box projection (deprecated) 6 ~35% Non-square
Tetrahedral (current) 4 ~12% Square 2×2

6. Phase 1 — Geometry Bake (Arnold AOV Extraction)

The plugin creates four Arnold camera nodes, positions them using the transforms from Eq. (3), and executes two render passes per camera:

  • Normals AOV (N): World-space surface normals encoded as RGB. Primary ControlNet conditioning input.
  • Depth AOV (Z): Per-pixel camera distance, tone-mapped to [0, 1]. Used for seam blending in Phase 3.

All renders use an aiFlat shading override to eliminate lighting variation. Output: 8 EXR files (4 views × 2 AOVs) composited into two 2×2 PNG atlases.

# Simplified extract — engine.py
def extract_geometry_passes(mesh, cameras, config):
    arnold.AiBegin()
    arnold.AiMsgSetConsoleFlags(arnold.AI_LOG_ERRORS)
    for i, cam in enumerate(cameras):
        _set_arnold_camera(cam)
        _render_aov("N", out=f"view_{i}_normals.exr")
        _render_aov("Z", out=f"view_{i}_depth.exr")
    arnold.AiEnd()

7. Phase 2 — AI Inference (Diffusion + ControlNet)

The normals atlas is passed as the ControlNet conditioning image. The model is invoked with a text prompt and generation parameters from rules.json. Output: a 1024×1024 RGB texture atlas.

Inference Parameters

Parameter Default Description
CFG scale 7.5 Higher = stronger prompt adherence.
ControlNet scale 0.85 Lower = more creative freedom vs. geometry fidelity.
Steps 30 Denoising steps. Increase for quality, reduce for speed.
Negative prompt See rules.json Concepts to suppress in the output.

The standard DDPM formulation applies. At each step $t$, the model predicts the noise component conditioned on the text embedding $\tau_\theta(y)$ and the ControlNet signal $c$:

(4) $$\hat{\epsilon} = \epsilon_\theta\!\left(\mathbf{z}_t,\, t,\, \tau_\theta(y),\, c\right)$$

Classifier-free guidance interpolates between the conditional and unconditional predictions:

(5) $$\tilde{\epsilon} = \epsilon_\theta(\mathbf{z}_t, t, \emptyset) + w \cdot \left[\epsilon_\theta(\mathbf{z}_t, t, \tau_\theta(y), c) - \epsilon_\theta(\mathbf{z}_t, t, \emptyset)\right]$$

where $w$ is the CFG scale.

8. Phase 3 — UV Reprojection

The four quadrants of the output atlas are mapped back to UV space using the stored UV manifest. At view-boundary seams, pixels from multiple views are blended by a per-pixel weight $w_i$ derived from the depth atlas:

(6) $$\mathbf{C}(u,v) = \frac{\displaystyle\sum_{i} w_i(u,v)\cdot \mathbf{C}_i(u,v)}{\displaystyle\sum_{i} w_i(u,v)}$$

where $\mathbf{C}_i(u,v)$ is the colour from view $i$'s quadrant and $w_i$ is inversely proportional to the normalised depth at that pixel (closer surface = higher weight).

The final texture is written as a 2K EXR and connected to a new aiStandardSurface shader. Colour space is set to Raw to prevent gamma correction.

UV quality: Reprojection accuracy depends on UV layout quality. Overlapping shells, very low texel density, or islands rotated more than ~45° from the nearest viewpoint will produce visible artifacts.

9. Dynamic Quantization

At startup, engine.py queries available VRAM via torch.cuda.mem_get_info() and selects a precision level automatically:

VRAM available Precision Load method Quality impact
≥ 16 GB FP16 Full model in VRAM None
8 – 15 GB INT8 bitsandbytes 8-bit Minor
4 – 7 GB INT4 bitsandbytes 4-bit Noticeable softening
< 4 GB / CPU INT4 + offload Accelerate CPU offload Testing only

All thresholds are configurable in config.yaml under quantization. FP16 can be forced regardless of detected VRAM via quantization.force_fp16: true.

10. Module — main.py

Maya-facing entry point. Handles plugin registration, the PySide2 UI, user input collection, and final shader application.

Function Description
initializePlugin(obj) Maya plugin entry point. Registers shelf button, loads config.
show_ui() Creates and shows the Automaytex panel. Callable from Script Editor.
on_generate() Validates selection, reads UI state, spawns worker thread, calls engine.run_pipeline().
apply_texture(path) Creates aiStandardSurface + file node, connects to mesh, sets colour space to Raw.
uninitializePlugin(obj) Cleans up on plugin unload.

A worker thread is used so that diffusion inference (30–120 s) does not block Maya's UI event loop. Progress is communicated back to the main thread via Qt signals.

11. Module — engine.py

All pipeline logic independent of Maya's UI. Can be imported and unit-tested from a standalone Python environment.

Function Description
run_pipeline(mesh, prompt, model, config) Top-level orchestrator. Calls all phases in sequence.
build_camera_matrix(direction, centroid, r) Returns Arnold camera world-space transform from a tetrahedral vertex direction.
extract_geometry_passes(mesh, cameras, config) Triggers Arnold renders; saves Normals and Depth EXRs to temp dir.
build_atlas(exr_paths) Composites 4 EXR passes into a 2×2 PNG atlas.
detect_vram() Returns available VRAM in GB; falls back to RAM estimate on CPU.
run_diffusion(atlas, prompt, config) Loads model at correct precision, runs inference, returns PIL image.
reproject_to_uv(result, uv_manifest, depth_atlas) Maps AI output to UV space with depth-weighted blending (Eq. 6).

16. AI Usage & Validation

Where AI Assistance Was Applied

  • Arnold API boilerplate: LLM assistance was used for AiRender() session management and AOV output driver configuration, which are sparsely documented for Python bindings.
  • Camera rotation matrices: Initial tetrahedral transform code was LLM-drafted, then verified analytically against known tetrahedron vertex positions.
  • ControlNet parameter defaults: Initial CFG and ControlNet scale values were LLM-suggested, then tuned by hand across a test set of 20 reference meshes.

Manual Validation Protocol

  • Hypergraph audit: Every shader node and projection network was inspected in Maya's Hypergraph editor to confirm correct connections and attribute values.
  • Geometry edge fidelity: ControlNet weights were tuned on hard-surface meshes to ensure edges remained sharp and free of diffusion-induced blurring.
  • Seam inspection: All four view boundaries inspected at 4K across the 20-mesh test set. Blending weights adjusted until no hard seams were visible under directional lighting.
Statement: The core pipeline architecture, tetrahedral strategy, and UV reprojection algorithm were designed by the author. No AI-generated code was integrated without manual review and, where applicable, analytical verification.

17. Troubleshooting

Plugin fails to load in Maya

Verify dependencies are installed via mayapy, not the system Python. Test: mayapy -c "import torch; print(torch.__version__)".

CUDA out of memory during inference

Reduce render.atlas_resolution to 512 in config.yaml, or lower the VRAM thresholds to force INT4. Restart Maya after editing config.

Visible seams in the reprojected texture

Reorganise the UV layout so each island is primarily visible from one tetrahedral viewpoint. Increase render.aov_samples to improve depth-pass accuracy.

Arnold renders black or empty AOVs

Open the Arnold RenderView manually and confirm renders succeed. Check that no render layer override is suppressing the aiFlat shader.

Generated texture does not match the prompt

Increase cfg_scale in rules.json (try 8–12). If geometry is well-captured but surface is generic, reduce controlnet_scale from 0.85 toward 0.70.

18. Glossary

Term Definition
AOV Arbitrary Output Variable. A named Arnold render pass capturing a specific shading component (normals, depth) as a separate image.
Atlas A single image containing multiple sub-images in a grid. Used here to present all 4 tetrahedral views to the Diffusion Model as one square input.
ControlNet A conditioning mechanism that anchors a Diffusion Model's spatial output to a guide image (normals, depth, edge map).
CFG scale Classifier-Free Guidance scale. Controls prompt adherence versus sampling diversity.
Diffusion Model A generative AI model that learns to reverse a noise process to produce structured images. SDXL, Flux, and SD1.5 are all Diffusion Models.
FP16 / INT8 / INT4 Weight precision levels. Lower precision reduces VRAM usage with minor quality trade-offs.
Hypergraph Maya's node graph editor for inspecting and modifying shader networks.
MtoA Maya to Arnold. The official Arnold plugin for Maya, exposing Arnold's Python API.
Reprojection Mapping a texture generated in camera space back onto a mesh's UV layout.
Tetrahedron A solid with 4 triangular faces. A regular tetrahedron inscribed in a sphere provides 4 viewpoints separated by $\approx 109.47°$.
UV / UV Map A 2D parameterisation of a 3D surface defining how textures are applied to a mesh.
VRAM Video RAM — the dedicated GPU memory limiting the precision at which a model can be loaded.
